# zinx-threads

Slack-style team chat. **electron-vite (Vite + TypeScript)** desktop shell; **React 19** UI + **Convex** backend + **WorkOS AuthKit** auth (both scaffolded, not yet live — see `SETUP.md`). Electron 39. **pnpm, not npm.**

**One codebase → desktop AND web.** The `src/renderer` app is the product; the Electron main/preload are just the desktop shell. It also builds as a plain web app (`vite.web.config.ts`) because data goes straight to Convex (works anywhere), native calls go through `src/renderer/src/lib/platform.ts` (feature-detect + web fallback — renderer code must NOT touch `window.electron`/`window.api` directly), and `router.ts` picks history per target (`isElectron` → hash for `file://`; web → browser history). Keep the renderer web-portable: no Electron/Node imports in `src/renderer`, always go through the `platform` layer.

## Commands

- `pnpm dev` — desktop app with HMR (renderer) + hot-reload (main/preload)
- `pnpm dev:web` — the same renderer as a **web app** (Vite, http://localhost:5173)
- `pnpm build` — typecheck, then `electron-vite build` → `out/` (desktop)
- `pnpm build:web` — typecheck:web, then `vite build` → `out/web` (web) · `pnpm preview:web`
- `pnpm start` — preview a production desktop build (`electron-vite preview`)
- `pnpm lint` — ESLint · `pnpm typecheck` — tsc (node + web) · `pnpm format` — Prettier
- `pnpm build:win` — build + package installer via electron-builder (also `build:mac`, `build:linux`)

## Process model

Three isolated processes, one `electron.vite.config.ts` with a `main` / `preload` / `renderer` section each.

- `src/main/index.ts` — main process (Node): windows, app lifecycle, OS APIs. Uses `@electron-toolkit/utils`.
- `src/preload/index.ts` — bridge: expose a _narrow, typed_ API via `contextBridge.exposeInMainWorld`. No raw `ipcRenderer`, no Node, to the renderer. Types in `src/preload/index.d.ts` (augments `window.electron` / `window.api`).
- `src/renderer/src/` — renderer (browser): React UI only. `@renderer` → `src/renderer/src`. Talks to main via the preload API; talks to Convex directly. Entry `main.tsx`, HTML `src/renderer/index.html`.
- **Preload bridge** exposes one method so far — `api.openExternal(url)` → validated `open-external` `ipcMain.handle` in main (`http/https` only) → `shell.openExternal`. `platform.openExternal` uses it on desktop, `window.open` on web.

## Backend & auth (scaffolded, not live — see `SETUP.md`)

- **Data model + auth are chosen but not wired**: swap the mock getters in `data/workspaces.ts` for Convex, auth via **WorkOS AuthKit** (chosen over Better Auth/Clerk: it's the Convex-default integration, free to 1M MAU, enterprise SSO/SCIM bolt on **per production connection** later — no rewrite). Decision rationale in memory `auth-stack-workos`.
- `convex/` holds the backend scaffold (inert until `npx convex dev`; it's outside the app tsconfigs + eslint-ignored): `schema.ts` = **SSO-ready identity model** (`users` with `externalId`+`provider`, `workspaces.organizationId` = WorkOS org, `workspaceMembers.directoryUserId`+`active` for SCIM, `roles.groupKeys` for group→role); `auth.config.ts` = WorkOS `customJwt`; `convex.config.ts` = `@convex-dev/workos-authkit` component.
- **Keys**: `.env.sample` documents every var + where it goes — frontend `VITE_*` in `.env.local`, Convex secrets via `npx convex env set`. `.env.local` is git-ignored.
- **Electron auth caveat**: hosted redirect login needs a custom-protocol deep link (`zinx://callback`) on desktop; web needs none. Deferred until auth is turned on (see `SETUP.md`).
- **CSP** (`src/renderer/index.html`) is widened `connect-src` for Convex (`*.convex.cloud` wss/https) + WorkOS; applies to both targets.

## UI / renderer stack

- **Tailwind v4** via `@tailwindcss/vite` (in the renderer plugins). No config file — the theme lives in `src/renderer/src/assets/globals.css` (`@theme`, CSS variables). That is the single global stylesheet, imported by `main.tsx`.
- **shadcn/ui** — base is **Base UI** (`@base-ui/react`, _not_ Radix), style **`base-nova`**, icons **Phosphor** (`@phosphor-icons/react`). Config in `components.json`; components land in `src/renderer/src/components/ui/` and import via `@renderer/*`. **The full base-nova set is installed** (`shadcn add --all`) — ~60 components. Add more with `pnpm dlx shadcn@latest add <name>`. `cn()` helper in `src/renderer/src/lib/utils.ts`. ESLint relaxes vendored-code rules for `components/ui/**` (return-type, fast-refresh, `react-hooks/{set-state-in-effect,exhaustive-deps,refs}`, `react/prop-types`, `no-explicit-any`) — plus the shadcn `use-mobile` hook lives in `hooks/`.
- **dnd-kit** (`@dnd-kit/core`+`sortable`+`utilities`) — only for the **kanban**. The board primitive `components/ui/kanban.tsx` (`Kanban`/`KanbanBoard`/`KanbanColumn`/`KanbanColumnContent`/`KanbanColumnHandle`/`KanbanItem`/`KanbanItemHandle`/`KanbanOverlay`) is ported verbatim from `zinx-os` (`src/components/global/kanban.tsx`); it's Base-UI-render-based and handles all task/column DnD + the drag overlay, calling `onValueChange` with a `Record<columnId, Task[]>` (its key order = column order).
- **Zustand** — stores in `src/renderer/src/store/`.
- **BlockNote** (Notion-style page editor) — `@blocknote/core` + `@blocknote/react` + the **ariakit** variant (`@blocknote/ariakit`, chosen over `mantine`/`shadcn` so no extra UI lib is pulled in and it's neutral to theme). Lives in `components/app-shell/page-editor.tsx`, **lazy-loaded** (`React.lazy` in `channel-view`) so its ~2 MB ProseMirror chunk only loads for `page` channels. Its document is a **block array** → maps 1:1 onto the mock `PageBlock[]` (`pageToBlocks`) and later Convex. Native blocks cover heading/paragraph/bullet/numbered/todo(checkListItem)/quote/codeBlock/divider + table/image/video/audio/toggle (all free defaults, spread via `...defaultBlockSpecs`); **callout is a custom block** (`createReactBlockSpec`, added to the slash menu) — the template for future custom blocks. **Code blocks are Shiki-highlighted** — the default codeBlock spec is replaced with `createCodeBlockSpec(codeBlockOptions)` from `@blocknote/code-block` (Shiki **JS engine** + precompiled grammars, _no WASM_ → works under our `script-src 'self'` CSP; github-light/github-dark dual theme; each language grammar is its own lazy chunk, loaded on demand). **Multi-column is deliberately excluded** — `@blocknote/xl-multi-column` is `GPL-3.0 OR PROPRIETARY`, not free for a closed-source app. Slash menu is customized: items are sorted into a fixed group order (`SLASH_GROUP_ORDER`) so each group renders one header, and its `floatingUIOptions` replace BlockNote's `size` middleware with `offset+flip+shift` (+ `strategy:'fixed'`, `.bn-suggestion-menu` CSS max-height) so it **flips upward** near the page bottom instead of being clipped (needs `@floating-ui/react` as a direct dep). Themed to our tokens in `page-editor.css` (imported _after_ `@blocknote/ariakit/style.css`; overrides `--bn-colors-*`/font/radius **globally** on `.bn-root.bn-root` — menus portal to a body-level `.bn-root`, so scoping to the editor misses them — plus the ariakit variant's hardcoded `#007acc` on `.bn-ak-*`; base font is `1rem` on `.bn-default-styles` so pages honor the UI-scale). Light/dark passed via `theme` prop, derived from `theme-store` with `useSyncExternalStore`. **Notion-style page chrome** wraps the editor: a full-width **cover** (`page-cover.tsx` — change/reposition-drag/remove, gradient or image via `coverStyle` in `cover-data.ts`), an overlapping **emoji icon**, an editable **title**, and hover "Add icon / Add cover / Add comment" affordances. Pickers are centered modals: `emoji-picker.tsx` (curated `emoji-data.ts`) and `cover-picker.tsx` (gradient gallery + curated **Unsplash** search-filter + link tab — modeled on `_zinx`'s `unsplash-image-picker`, but a fixed gallery since there's no backend to hold an Unsplash key). Cover/icon/title are local `useState` (mock); data-file split (`cover-data.ts`/`emoji-data.ts`) keeps the picker files components-only for react-refresh. **CSP `img-src` was widened to `'self' data: https:`** so external covers (Unsplash / pasted links) load. Edits are in-editor only for now (no persistence yet).
- **Theming** — `store/theme-store.ts` toggles the `.dark` class on `<html>`, persists to `localStorage` (key `zinx-theme`), supports light / dark / **system**; **default `system`** (follows OS, live-updates). Resolved before render — the store is imported first in `main.tsx` (CSP `script-src 'self'` rules out an inline init script). Switcher UI: `components/theme-switcher.tsx` (header).
- **TanStack Router** — **file-based** routing. Routes in `src/renderer/src/routes/`; the `tanstackRouter()` Vite plugin generates `src/renderer/src/routeTree.gen.ts` on dev/build (generated — don't edit). `router.ts` creates the router directly with **hash history** (`createHashHistory` — the renderer loads over `file://` in production) and registers its type; `main.tsx` renders `<RouterProvider>`. Dev-only `<TanStackRouterDevtools>` lives in `routes/__root.tsx`. NB: the plugin's `routesDirectory`/`generatedRouteTree` resolve relative to the renderer root `src/renderer`, not the project root.

## App shell & routing

**NOT a Discord clone** (deliberate — see project memory `design-direction`). Shell in `components/app-shell/`: by default **no server rail** — orgs live in a `workspace-switcher` dropdown (avatar + name + "Workspace" subtitle, active item gets a left primary border — modeled on the `_zinx` community switcher) at the top of one unified sidebar. A Discord-style **`server-rail.tsx`** is an **opt-in** left column (settings toggle `settingsStore.showServerRail`, desktop-only — squircle→rounded avatars, left active pill, mention badge). → **nestable outline** (recursive `getSidebarTree`: groups can contain channels AND sub-groups; drag-and-drop reorder/move channels + groups via `store/sidebar-store.ts`, HTML5 DnD, in-session) + quick nav (search/inbox/threads) + user panel → message pane (`channel-view` = breadcrumb header + `message-list`/`message-item` + `message-composer`) → **right panel = members OR thread only** (everything else is a popup). Overlays: `settings-dialog`, `inbox-popover`, `threads-popover`, `command-palette`.

- **Resizable panes:** sidebar + right panel via `resize-handle.tsx`; widths persisted in `ui-store` (`sidebarWidth`/`rightWidth`/`threadWidth`, **clamped** — the clamp min is each pane's floor, e.g. sidebar ≥220, thread ≥320 — localStorage).
- **Responsive (≥`lg` / `<lg`):** below the `lg` breakpoint the sidebar becomes a **slide-in drawer** over the content (Slack/Notion-style) toggled by a **hamburger** in the channel header, with a scrim backdrop (`ui-store.navOpen`); the resize handle + inline pane width are desktop-only (gated by `useMediaQuery('(min-width:1024px)')`, `lib/use-media-query.ts`). On compact the inline right panel is hidden — a thread instead opens as a **right-side overlay** (member list is desktop-only). Selecting a channel closes the drawer.
- **Rename channels:** double-click a channel name (or the hover gear) in the sidebar to edit inline; the override lives in `store/channel-store.ts` (`names`, in-session) and is read via `useChannelName(id, fallback)` in the sidebar + header. (Command palette / breadcrumb still show mock names for now.)
- Routes (file-based): `/` → first server; `/w/$workspaceId` (`w.$workspaceId.tsx`) is the layout; `/w/$workspaceId/` → default channel; `/w/$workspaceId/c/$channelId` renders `ChannelView`. **URL = source of truth**. (Route param `workspaceId` carries a server id.)
- Panels/overlays are **UI state** in `store/ui-store.ts` (member list, active thread, reply, inbox, settings, threads, palette, pane widths), not routes.
- Active nav uses TanStack `activeProps`/`inactiveProps` (mutually exclusive → no Tailwind conflicts).
- **Features modeled:** cross-org **shared channels** (`channel.shared.withServerIds`, indicator in sidebar + header); **Discord-style threads** (first-class: thread indicator on messages, thread side panel, active-threads popover); **member list** grouped by hoisted role + presence; replies; notifications inbox; **4-level UI scale** (`store/settings-store.ts` sets the root `html` font-size — 12/14/16/18px — so the whole rem-based UI zooms, not just chat text). Panel widths are px and don't scale (independently resizable).
- **Channel `kind` = `chat | voice | page | kanban`.** `page` channels render a **Notion-style block editor** (`page-editor.tsx`, BlockNote — see UI stack); `kanban` channels render a **project board** — **ported 1:1 from `zinx-os`** (`D:\Projects\zinx-os\src\components\kanban`) into `components/kanban/` (`board-view.tsx`/`task-board.tsx`/`task-card.tsx`/`priority-badge.tsx`) on the **dnd-kit `Kanban` primitive** (`components/ui/kanban.tsx`) + shadcn `Card`/`Badge`/`Avatar`/`Button`. Cards: flag priority badge / labels / assignee (avatar) / due-date-urgency / points, `border-l-primary` for tasks assigned to you, `hover:shadow-md`; a card and the column `+ Add task` both open one **create/edit task dialog** (`kanban/task-dialog.tsx`, `mode: 'create' | 'edit'` — `DialogContent` is `flex max-h-[90dvh] flex-col sm:max-w-2xl` with a header (title + subtitle: create→`Adding to "<column>"`, edit→`Update task details`), a scrollable `min-h-0 flex-1 overflow-y-auto` body, and a footer (`sm:justify-between`: Delete [edit-only, `text-destructive`] · Cancel · Create/Save; the primary button is `type="submit" form="task-form"`). Fields: title, **markdown description** (`markdown-textarea.tsx` — Write/Preview tabs + toolbar + char count, preview shows raw text since there's no Convex @/# autocomplete), **checklist** (`task-checklist-field.tsx` — progress bar + per-item checkbox with a display/edit textarea toggle + delete + Add item), `Select` (priority), **multi-assignee** `Popover`/`Command` picker (removable avatar chips), `Calendar` (due date), label chips. Ported from zinx-os's create-task/edit-task-dialog + task-form-fields; **comments + story-points editing intentionally omitted for now** (story points are preserved on edit but not editable). `KanbanTask` carries `assigneeIds: string[]` (avatar stack on cards) + `checklist: ChecklistItem[]` (done/total badge on cards). dnd-kit handles moving **cards + columns** (with a drag overlay ghost); columns are add/rename(double-click)/delete-able; `+ Add column`. State is local `useState` (`tasksByColumn` + `names`, keyed by id); data via `getBoard`/`Board`/`KanbanColumn`/`KanbanTask`) instead of the message view; `voice` are non-navigating; `chat` is the message view.
- **Data is mock**: `data/workspaces.ts` — getters `getServer/getChannels/getChannel/getMembers/getMessages/getThread/getThreadsForServer/getNotifications/getPage/getBoard`. Swap for Convex; components consume by function so the change is localized. Composer sends + page todos + kanban card moves/adds are local `useState` only.
- **Visual system:** **edge-to-edge** — the app fills the window (no outer gutter/rounded-card; a floating frame is redundant inside the OS window and the app is used windowed). Depth is internal: darker `bg-sidebar` nav vs lighter `bg-card` message surface, divided by borders + the resize handle. Header is minimal (channel name + shared badge + search/threads/inbox/members). Scrollbars thin, theme-aware, reveal-on-hover (globals.css) **on the main app surfaces only** — **dialogs / modals / overlays scroll with the scrollbar hidden** (globals.css targets the shadcn `data-slot` popups: dialog/alert-dialog/sheet/drawer/popover/select/command + descendants; custom overlays — command palette, settings, emoji/cover pickers, inbox & threads popovers — opt in with the `.no-scrollbar` utility). (`--canvas` token retained but currently unused.)
- **⌘K command palette** (`command-palette.tsx`, `ui-store.paletteOpen`, global listener in the layout): jump to channels / threads / people / servers / actions with keyboard nav. The channel-header search opens it.
- **Deferred:** a frameless custom title bar (window min/max/close) — needs `frame:false` + main/preload IPC and GUI verification; not built yet.

## Non-negotiables

- **Security:** keep `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true` (set explicitly in `src/main/index.ts` — the electron-vite template ships `sandbox: false`; **we override it**, don't revert). Validate every IPC input in main. Never put secrets/API keys in the renderer — proxy through Convex or main.
- Main ↔ renderer only via `contextBridge` + `ipcMain.handle` / `ipcRenderer.invoke`. Keep `setWindowOpenHandler` denying new windows.
- Don't add `openDevTools()` to shipped code (F12 in dev is fine — handled by `@electron-toolkit/utils` `optimizer`).

## pnpm / packaging notes

- **Build-script allow-list lives in `pnpm-workspace.yaml`** (`allowBuilds:` — electron, esbuild, electron-winstaller). pnpm 11 no longer reads the `pnpm` field in `package.json`; if esbuild/electron native binaries fail to build, that's why.
- `.npmrc` needs `shamefully-hoist=true` for electron-builder to resolve hoisted deps. (The scaffold's China-mirror lines were removed; Electron binaries download from the official GitHub source.)
- **Windows gotcha on this machine:** `pnpm add` / `pnpm remove` fail with `EPERM: … rename … package.json` (something locks the file). Workaround: hand-edit `package.json`, then run `pnpm install` (which doesn't rewrite it) to sync `node_modules` + lockfile.
- Packaging config: `electron-builder.yml`. Auto-update wired via `electron-updater` + `dev-app-update.yml` (present, not yet hooked into main).
